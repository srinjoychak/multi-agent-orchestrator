# Argue

Use this skill for design-heavy decisions before writing code.

## Workflow

1. Write the initial design in `DESIGN.md`.
2. Critique the design with a second pass.
3. Revise the document to address findings.
4. Repeat until the design is stable or the round limit is reached.

## Rules

- Keep `DESIGN.md` as a clean design doc.
- Do not mix implementation code into the debate.
- Address every finding explicitly.
